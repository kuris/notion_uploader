#### 1. 📈 **진행률 시각화 강화**

✅ 퍼센트 표시  
✅ 남은 시간 자동 계산  
✅ 처리 속도 실시간 표시  

#### 2. ⚡ **글길이 바이탈 사인 (Vital Signs)**
```
심박계처럼 글자 수를 바이탈 사인으로 시각화

████ ██ ██████████ ████ ██
🔵   🟢 🔵        🟢   🔴
(처리)(성공)(진행중)(성공)(실패)
```

✅ 각 파일의 글자 수 시각화  
✅ 처리 상태별 색상 구분  
✅ 호버 시 상세 정보 표시  

#### 3. 🎯 **탭 시스템 (Tab System)**
```
[📊 통계]  [⚠️ 실패 (3)]

Tab 1: 통계
├─ 성공: 1,234
├─ 건너뜀: 2  ℹ️ 같은 제목 페이지 존재 시 자동 건너뜀
└─ 실패: 3

Tab 2: 실패한 파일 목록
├─ sample1.md
│  ❌ API 제한 (429)
│  [🔄 재시도]
├─ sample2.md
│  ❌ 파일 손상
│  [🔄 재시도]
```

✅ 통계/실패 탭 분리  
✅ 개별 파일 재시도 가능  
✅ 상세 에러 메시지  

#### 4. 💡 **사용자 가이드 강화**
```
🔷 상단 설명
"마크다운 폴더를 선택하면, AI가 요약/태그를 생성하여 
 노션 DB로 자동 업로드합니다. 대량 처리 시 중복 생성을 방지합니다."

🔷 건너뜀 기준 툴팁
건너뜀: 2  ℹ️
        └─ "같은 제목의 페이지가 이미 존재할 때 자동으로 건너뜁니다"
```

✅ 상단 설명 추가  
✅ 인터랙티브 툴팁  
✅ 멱등성 개념 설명  

#### 5. 🔄 **API Rate Limit 자동 대응 (지수 백오프)**
```javascript
// server.js에 추가된 지수 백오프
HTTP 429 (Too Many Requests)
  ↓
자동 감지 및 재시도 (1초 대기)
  ↓
여전히 실패 시 (2초 대기)
  ↓
마지막 시도 (4초 대기)
  ↓
3회 재시도 후에도 실패 → 사용자에게 에러 표시
```

✅ HTTP 429/503 자동 감지  
✅ 지수 백오프: 1초 → 2초 → 4초  
✅ 최대 3회 재시도

## 🎯 핵심 개선사항 3가지

### 1️⃣ **멱등성(Idempotency) 완벽 보장** ⭐
```
첫 번째 실행
├─ 1,990개 파일 선택
├─ AI 요약 생성 (500초)
├─ Notion DB 업로드 (330초)
└─ ✅ 1,990페이지 생성

두 번째 실행 (같은 폴더, 같은 파일)
├─ 1,990개 파일 선택
├─ notion_page_id 감지 → 자동 건너뜀 ⏭️
└─ ✅ 모두 스킵 (중복 없음!)

또는 Toggle ON: "이미 업로드된 글 요약 채우기"
├─ 1,990개 파일 선택
├─ 기존 요약만 업데이트
└─ ✅ 안전한 부분 업데이트
```

### 2️⃣ **실시간 모니터링으로 사용자 경험 극대화** ⭐
```
┌─ 87% 진행 (1,234 / 1,412)
├─ ⚡ 22개/분 처리 중
├─ ⏳ 약 8분 45초 남음
└─ 📈 바이탈 그래프로 각 파일 상태 한눈에 파악
```

사용자가 언제든 "지금 몇 개나 됐지?" "언제 끝날까?"에 답할 수 있습니다.

### 3️⃣ **자동 에러 복구로 불안감 제거** ⭐
```
API 429 에러 발생
  ↓
자동 재시도 (사용자 개입 없음)
  ↓
여전히 실패 시 → 탭에 표시 + 개별 재시도 버튼
  ↓
사용자가 시간 후 클릭 → 안전하게 재처리
```

## ⏱️ 1,990개 파일 처리 예상 시간

### 성능 계산 (상세)

| 항목 | 계산식 | 소요 시간 |
|------|--------|---------|
| 순수 Notion API | 1,990 ÷ 3 req/sec | 11분 |
| AI 요약 생성 (병렬) | 1,990 × 3초 평균 | 1시간 |
| 네트워크 지연 | 1,990 × 0.5초 | 17분 |
| 재시도 (예상 2% 실패) | 1,990 × 0.02 × 5초 | 2분 |
| **합계** | | **1시간 10분~1시간 40분** |

### Rate Limit 자동 처리로 +2~3분 추가 (무시할 수준)

```
시나리오: API 5% 429 에러 발생
├─ 99.5개 파일 × 3회 재시도
├─ 1회차: 1초 대기 → 성공
├─ 2회차: 2초 대기 → 성공  
├─ 3회차: 4초 대기 → 성공
└─ 추가 시간: 약 2분 30초 (무시할 수준)
```

## 🔒 안전성 검증

### ✅ 중복 생성 불가능
```javascript
YAML 프론트매터에서 notion_page_id 검사
├─ 있음 + Toggle OFF → 건너뜀
├─ 있음 + Toggle ON → 요약만 업데이트
└─ 없음 → 새 페이지 생성
```

### ✅ 부분 실패 후 복구
```
1,990개 중 500개 처리 중 오류 발생
├─ 처리 완료된 500개: 변경 없음
├─ 처리 중: 상태 저장
└─ 500~1,990: 아직 처리 안 됨

재시도 클릭
├─ 성공한 500개: 건너뜀 (멱등성)
├─ 실패한 나머지: 다시 시도
└─ 최종: 1,990개 모두 처리 완료
```

### ✅ Rate Limit 자동 처리
```
HTTP 429 Too Many Requests
├─ 1회차: 1초 대기 후 재시도
├─ 2회차: 2초 대기 후 재시도
├─ 3회차: 4초 대기 후 재시도
└─ 3회 모두 실패 시: 사용자 정보 제공 후 수동 재시도 가능
```
